# nix-config

Personal NixOS configuration.

## Hosts

| Host       | Device        | Arch            | Role                                  |
| ---------- | ------------- | --------------- | ------------------------------------- |
| `desuwa`   | Desktop       | `x86_64-linux`  | Main workstation                      |
| `kamo`     | ROG Ally      | `x86_64-linux`  | Portable / docked development machine |
| `dane`     | Radxa Rock 5T | `aarch64-linux` | Server / always-on development helper |
| `nanodesu` | Laptop        | `x86_64-linux`  | Laptop                                |

## Layout

```text
hosts/
  <hostname>/
    default.nix
    disk.nix                         # generated by scripts/install-host
    hardware-configuration.nix        # generated by scripts/install-host
    ssh_host_ed25519_key.pub          # generated by scripts/install-host
    home/
      default.nix

modules/
  core/
  desktop/
  flake/
  home/
  profiles/
  security/
  storage/
  theme/

secrets/
  keys.nix
  secrets.nix
```

Host-specific machine data should live under `hosts/<hostname>/`.

- `disk.nix` contains the stable `/dev/disk/by-id/...` install target.
- `hardware-configuration.nix` is generated from the NixOS ISO environment.
- `ssh_host_ed25519_key.pub` is the target host's SSH public host key used by agenix.

Private SSH host keys are never committed. They are copied temporarily to `/tmp/<hostname>/etc/ssh` for `nixos-anywhere --extra-files`.

## Install a host

Boot the official NixOS ISO on the target machine.

On the target:

```bash
# Connect to Wi-Fi if needed
nmtui

# Set a password for the nixos user
passwd

# Show the IP address
ip addr
```

From this repository:

```bash
./install-host kamo nixos@192.168.0.99
```

Or pass the disk path explicitly:

```bash
./install-host kamo nixos@192.168.0.99 /dev/disk/by-id/nvme-...
```

The installer script handles:

- SSH connectivity check
- disk ID selection
- `hosts/<host>/disk.nix` generation
- `hosts/<host>/hardware-configuration.nix` generation
- `hosts/<host>/ssh_host_ed25519_key.pub` generation
- `/tmp/<host>/etc/ssh` preparation for `nixos-anywhere --extra-files`
- `agenix rekey`
- `nix fmt`
- host evaluation
- `nixos-anywhere`

Do not SSH as root. Use the `nixos` user from the ISO.

## Required `secrets/keys.nix` shape

`secrets/keys.nix` should read host public keys from `hosts/<host>/ssh_host_ed25519_key.pub`.

Example:

```nix
let
  readKey = path: builtins.replaceStrings [ "\n" ] [ "" ] (builtins.readFile path);
in
{
  users = {
    yawarakatai = "ssh-ed25519 ...";
  };

  hosts = {
    desuwa = readKey ../hosts/desuwa/ssh_host_ed25519_key.pub;
    kamo = readKey ../hosts/kamo/ssh_host_ed25519_key.pub;
    dane = readKey ../hosts/dane/ssh_host_ed25519_key.pub;
    nanodesu = readKey ../hosts/nanodesu/ssh_host_ed25519_key.pub;
  };
}
```

The install script only writes the `.pub` file under the host directory. `keys.nix` should not need to be edited for every reinstall once this structure is in place.

## Required host `disk.nix` usage

Host configs should read the generated disk file.

Example:

```nix
# hosts/kamo/default.nix
{ ... }:

let
  disk = import ./disk.nix;
in
{
  imports = [
    (import ../../modules/storage/disko-ext4.nix {
      device = disk.device;
    })

    ./hardware-configuration.nix
  ];
}
```

## Notes

### SSH host keys

The SSH host public key is used for agenix secrets. The same ISO host key is also preserved into the installed system through `nixos-anywhere --extra-files`.

This avoids a mismatch between:

1. `hosts/<host>/ssh_host_ed25519_key.pub`
2. encrypted agenix recipients
3. installed `/etc/ssh/ssh_host_ed25519_key`

### Root SSH

The normal install flow uses `nixos@<target-ip>`. The script uses `sudo` only for reading the private ISO host key when preparing `--extra-files`.

### Lanzaboote / TPM2

The default install flow does not use lanzaboote or TPM2 unlock setup.

### Filesystems

Use the host's selected disko module. For simple single-boot machines, ext4 is preferred unless a host specifically needs Btrfs features such as snapshots, send/receive, or compression.

### Kernel

Do not set `linuxPackages_latest` globally by default.

Prefer hardware-specific modules first. For example, `nixos-hardware.nixosModules.asus-ally-rc71l` should be used for `kamo`. Add an explicit `boot.kernelPackages = pkgs.linuxPackages_latest;` only for hosts that actually need newer kernel behavior.
