# nix-config

Personal NixOS configuration.

## Hosts

| Host       | Device        | Arch            | Role                                  |
| ---------- | ------------- | --------------- | ------------------------------------- |
| `desuwa`   | Desktop       | `x86_64-linux`  | Main workstation                      |
| `kamo`     | ROG Ally      | `x86_64-linux`  | Portable / docked development machine |
| `dane`     | Radxa Rock 5T | `aarch64-linux` | Server / always-on development helper |
| `nanodesu` | Laptop        | `x86_64-linux`  | Laptop                                |

## Layout

```text
hosts/
  <hostname>/
    default.nix                       # import glue only
    hardware.nix                      # host-specific hardware modules
    system.nix                        # host-specific handwritten settings
    storage/
      disko.nix                       # host-owned install disk layout
      default.nix                     # optional runtime storage settings
    generated/
      hardware-configuration.nix      # generated by install-host
      ssh_host_ed25519_key.pub        # generated by install-host
    home/
      default.nix

modules/
  core/
  desktop/
  flake/
  home/
  profiles/
  security/
  storage/
  theme/

secrets/
  keys.nix
  secrets.nix
```

Host-specific machine data should live under `hosts/<hostname>/`.

- `storage/disko.nix` contains the host's stable `/dev/disk/by-id/...` install disk layout.
- `generated/hardware-configuration.nix` is generated from the NixOS ISO environment.
- `generated/ssh_host_ed25519_key.pub` is the target host's SSH public host key used by agenix.
- `default.nix` should stay close to import glue; put handwritten host settings in `system.nix`, `hardware.nix`, or `storage/`.

Private SSH host keys are never committed. They are copied temporarily to `/tmp/<hostname>/etc/ssh` for `nixos-anywhere --extra-files`.

## Where to edit

| Change | Edit |
| ------ | ---- |
| Host-specific boot, services, display, wallpaper, or package settings | `hosts/<host>/system.nix` |
| Host-specific hardware modules and device quirks | `hosts/<host>/hardware.nix` |
| Host install disk layout | `hosts/<host>/storage/disko.nix` |
| Generated hardware facts or SSH host public key | `hosts/<host>/generated/` via `install-host` |
| Home Manager host-only additions | `hosts/<host>/home/default.nix` |
| Reusable NixOS or Home Manager features | `modules/` |
| Profile aggregation | `modules/profiles/` |
| Published flake module names | `modules/flake/module-registry.nix` |

Display outputs are configured with `my.display.outputs`, keyed by output name. Set `enable = false` on an output to keep a known external display disabled.

Wallpaper is configured with `my.wallpaper.image` and is shared across all displays. When it is `null`, Niri uses `my.wallpaper.fallbackColor` as a solid background. Noctalia owns wallpaper JSON generation; Stylix should not set `stylix.image`.

## Install a host

Boot the official NixOS ISO on the target machine.

On the target:

```bash
# Connect to Wi-Fi if needed
nmtui

# Set a password for the nixos user
passwd

# Show the IP address
ip addr
```

Before running the installer, review `hosts/<host>/storage/disko.nix` and make sure the disk paths match the target machine.

From this repository:

```bash
./install-host kamo nixos@192.168.0.99
```

The installer script handles:

- SSH connectivity check
- remote disk ID display for comparison with `storage/disko.nix`
- `hosts/<host>/generated/hardware-configuration.nix` generation
- `hosts/<host>/generated/ssh_host_ed25519_key.pub` generation
- `/tmp/<host>/etc/ssh` preparation for `nixos-anywhere --extra-files`
- `agenix rekey`
- `nix fmt`
- host evaluation
- `nixos-anywhere`

Do not SSH as root. Use the `nixos` user from the ISO.

## Required `secrets/keys.nix` shape

`secrets/keys.nix` should read host public keys from `hosts/<host>/generated/ssh_host_ed25519_key.pub`.

Example:

```nix
let
  readKey = path: builtins.replaceStrings [ "\n" ] [ "" ] (builtins.readFile path);
in
{
  users = {
    yawarakatai = "ssh-ed25519 ...";
  };

  hosts = {
    desuwa = readKey ../hosts/desuwa/generated/ssh_host_ed25519_key.pub;
    kamo = readKey ../hosts/kamo/generated/ssh_host_ed25519_key.pub;
    dane = readKey ../hosts/dane/generated/ssh_host_ed25519_key.pub;
    nanodesu = readKey ../hosts/nanodesu/generated/ssh_host_ed25519_key.pub;
  };
}
```

The install script only writes the `.pub` file under `generated/`. `keys.nix` should not need to be edited for every reinstall once this structure is in place.

## Required host storage layout

Host configs should import their host-owned storage file directly.

Example:

```nix
# hosts/kamo/default.nix
{ ... }:

{
  imports = [
    ./storage/disko.nix
    ./generated/hardware-configuration.nix
    ./hardware.nix
    ./system.nix
  ];
}
```

## Notes

### SSH host keys

The SSH host public key is used for agenix secrets. The same ISO host key is also preserved into the installed system through `nixos-anywhere --extra-files`.

This avoids a mismatch between:

1. `hosts/<host>/generated/ssh_host_ed25519_key.pub`
2. encrypted agenix recipients
3. installed `/etc/ssh/ssh_host_ed25519_key`

### Root SSH

The normal install flow uses `nixos@<target-ip>`. The script uses `sudo` only for reading the private ISO host key when preparing `--extra-files`.

### Lanzaboote / TPM2

The default install flow does not use lanzaboote or TPM2 unlock setup.

### Filesystems

Use the host's `storage/disko.nix`. Shared disko helpers are intentionally avoided so each host's destructive disk layout is readable in one place. For simple single-boot machines, ext4 is preferred unless a host specifically needs Btrfs features such as snapshots, send/receive, or compression.

### Kernel

Do not set `linuxPackages_latest` globally by default.

Prefer hardware-specific modules first. For example, `nixos-hardware.nixosModules.asus-ally-rc71l` should be used for `kamo`. Add an explicit `boot.kernelPackages = pkgs.linuxPackages_latest;` only for hosts that actually need newer kernel behavior.
